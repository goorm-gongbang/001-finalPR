# 부하테스트 전용

부하 테스트 관측 화면은 Staging에서 진행한 부하 테스트 시나리오와 인프라 상태를 함께 확인하는 기준입니다. k6 결과와 Pod, Node, Istio 라우팅, 로그를 같이 보며 병목 구간과 확장 동작을 확인합니다.

---

## 주요 대시보드

| 대시보드 | 주요 지표 | 목적 |
|---|---|---|
| **k6 Load Test Combined** | 요청 수, 응답 시간, 오류율, 비즈니스 성공 지표 | 테스트 구간의 전체 결과 확인 |
| **k6 Load Test Nodes** | 노드 상태, 리소스 사용량, 역할별 부하 분포 | 노드 단위 병목 확인 |
| **k6 Load Test Webs Resources** | 웹 계층 Pod 리소스 사용량, 재시작, 스케일링 | 웹 계층 확장 동작 확인 |
| **k6 Load Test Diagnostics** | 테스트 구간 진단 지표 | 병목과 이상 징후 상세 확인 |
| **k6 Load Test Logs** | 테스트 중 발생한 로그와 예외 | 시나리오와 오류 구간 대조 |
| **k6 Istio Routing** | 라우팅 흐름과 트래픽 분산 | Istio 경로와 서비스 분배 확인 |

## 관측 기준

| 구분 | 확인 지표 | 목적 |
|---|---|---|
| **요청 처리량** | 요청 수, 응답 시간, 오류율 | 시나리오 전체 결과 확인 |
| **노드 상태** | CPU, 메모리, 역할별 노드 분포 | 자원 부족과 노드 병목 확인 |
| **웹 계층 상태** | Pod 리소스 사용량, 재시작, 스케일링 | 웹 계층 확장 동작 확인 |
| **Istio 경로** | 서비스 분배, 요청량, 오류율, 지연시간 | 메쉬 라우팅 상태 확인 |
| **테스트 로그** | 로그와 예외 메시지 | 시나리오와 장애 구간 대조 |

---

## 운영 기준

Staging 부하 테스트는 예매 오픈 구간을 가정한 시나리오를 기준으로 진행합니다. 테스트 구간에서는 응답 시간, 오류율, 스케일링 동작, 로그, Istio 라우팅을 함께 확인해 병목 구간을 판단합니다.

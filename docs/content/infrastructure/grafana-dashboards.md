# Grafana 대시보드

Playball은 서비스 상태, 인프라 상태, 보안 이벤트, 비즈니스 지표를 Grafana 대시보드 기준으로 확인합니다. 메트릭, 로그, 트레이스를 한 경로에서 연결해 보고, 운영 목적에 따라 대시보드 폴더를 구분해 사용합니다.

---

## 대시보드 구성 기준

| 구분 | 주요 내용 | 목적 |
|---|---|---|
| **서비스 관측** | 애플리케이션 응답 시간, 오류율, 인증 실패, 트래픽 흐름 | 서비스 상태와 배포 직후 변화 확인 |
| **인프라 관측** | 클러스터 상태, 노드 용량, 오토스케일링, Istio 메쉬 상태 | 인프라 자원 상태와 플랫폼 운영 확인 |
| **Security Ops** | WAF 차단, Rate Limit, 정책 위반, 보안 이벤트 | 보안 경로와 차단 이벤트 확인 |
| **비즈니스 KPI** | Hold 성공률, 주문 퍼널, 결제 성공률 | 티켓팅 흐름과 전환 지표 확인 |
| **사용자 행동 분석** | 프론트엔드 이벤트, 사용자 경로, 화면 전환 | 사용자 행동과 화면 흐름 분석 |

---

## 접근 권한 기준

| 항목 | 기준 |
|---|---|
| **로그인 방식** | Google OAuth 기반 로그인만 허용 |
| **기본 권한** | 로그인 사용자는 기본 `Editor` 권한으로 접근 |
| **운영 관리 권한** | 지정 계정은 `Admin` 권한으로 폴더와 대시보드 관리 |
| **권한 분리 목적** | 운영 확인은 넓게 열고, 대시보드 수정과 관리 권한은 제한 |
| **폴더 구조** | 운영 목적별 폴더로 분리해 필요한 화면을 빠르게 찾도록 구성 |

---

## 인프라 관측 주요 화면

### EKS 클러스터 개요

클러스터 전반의 Pod 수, 네임스페이스 수, 리소스 사용률, 노드 상태를 한 화면에서 확인합니다.

![EKS 클러스터 개요](/images/infrastructure/grafana-dashboards/eks-cluster-overview.png)

### K8s 운영 현황판

Ready 상태, 대기 중 Pod, CrashLoop, 리소스 사용량, Pod/Node 목록을 기준으로 배포 직후 상태와 장애 징후를 확인합니다.

![K8s 운영 현황판 (k9s 스타일)](/images/infrastructure/grafana-dashboards/k8s-ops-k9s.png)

### Istio 워크로드 모니터링

요청량, 오류율, 지연시간, TCP 트래픽을 기준으로 서비스 메쉬 내부의 트래픽 흐름과 오류 징후를 확인합니다.

![Istio 워크로드 모니터링](/images/infrastructure/grafana-dashboards/istio-workload-monitoring.png)

---

## 로그 / 트레이스 탐색

### Tempo Explore

Trace 탐색 화면에서 서비스 호출 순서와 지연 구간을 확인하고, 느린 요청이나 병목 구간을 추적합니다.

![Tempo Explore](/images/infrastructure/grafana-dashboards/explore-tempo.png)

## 운영 기준

| 구분 | 확인 경로 | 목적 |
|---|---|---|
| **서비스 상태 확인** | 서비스 관측 대시보드 | 5xx, 응답 시간, 인증 실패 확인 |
| **인프라 상태 확인** | 인프라 관측 대시보드 | Node, Pod, 오토스케일링, 리소스 사용량 확인 |
| **보안 이벤트 확인** | Security Ops 대시보드 | WAF 차단, Rate Limit, 정책 위반 확인 |
| **트레이스 확인** | Tempo Explore | 느린 요청, 서비스 간 호출 흐름 확인 |
| **비즈니스 지표 확인** | 비즈니스 KPI, 사용자 행동 분석 대시보드 | 티켓팅 흐름과 사용자 행동 분석 |

## 8장 — "공격과 방어를 동시에 만든다"

### 좌측 패널

| 위치 | 현재 | 수정 |
|------|------|------|
| 빨간 박스 제목 | T3 + 공격은 반대로 | **AI 매크로 공격은 반대로** |
| 초록 박스 2번째 | 에이전트 = 결정론 (LLM 없음) | **실행 → 결정론 상태머신 (재현 가능)** |
| 초록 박스 3번째 | 스웜 전략 = LLM (비동기 적응) | **전략 → LLM 비동기 조정** |

### 우측 패널 — 표

| 위치 | 현재 | 수정 |
|------|------|------|
| AI 공격 에이전트 · 역할 | 휴먼 모방 자동 예매 | **티켓팅 숙련자 모방 자동 예매** |
| AI Defense · 역할 | 실시간 탐지 + 사후 판단 | **실시간 탐지 + LLM 사후 판단 + 방어 정책 개선** |
| AI Defense · 스택 | D0 런타임 / 오프라인 LLM | **FastAPI + Redis + ClickHouse** |

### 수정 이유

- **"T3+"** 는 앞선 장표의 방어 등급 용어 — 여기선 **"AI 매크로"** 가 청중에게 직관적
- **"LLM 없음"** 은 "AI 안 쓰나?"로 오해 가능 — 실행/전략 분리 구조를 **긍정형**으로 표현
- **D0 런타임 · 오프라인 LLM** 은 아키텍처 설명이지 스택이 아님 — **실제 기술 스택**으로 교체

## 8장 — "공격 에이전트 구성"

### 우측 패널

| 위치 | 현재 | 수정 |
|------|------|------|
| 제목 | 스웜 구조 | **스웜 — 에이전트 팀 운영 구조** |
| 설명문 | 에이전트들의 이벤트는 LLM 코디네이터로 전달되고, 코디네이터는 스웜 전체의 구역 전략을 비동기로 조정 | **여러 에이전트가 독립적으로 예매를 시도하고, 각자의 결과(성공·차단·퀴즈)가 LLM 코디네이터로 모입니다. 코디네이터는 이를 보고 전체 구역 전략을 비동기로 조정합니다.** |

### 다이어그램

| 위치 | 현재 | 수정 |
|------|------|------|
| 상단 라벨 | 에이전트 (결정론 · 최대 10) | **에이전트 (결정론)** |
| 세 번째 박스 | Agent 3 | **Agent N** |

### 수정 이유

- "스웜"이란 단어를 그대로 쓰되, 바로 옆에 **"에이전트 팀 운영"** 풀이를 붙여 모르는 청중도 자연스럽게 이해
- 에이전트 수는 고정이 아니라 자유 — "최대 10" 제거, Agent 3 → **Agent N**

## 9-1장 — "AI 방어 : 실시간 탐지"

### 좌측 패널

| 위치 | 현재 | 수정 |
|------|------|------|
| 소제목 | 원칙 | **핵심 동작 방식** |

### 행동 지표 (7개 → 5개)

| 현재 (영문만, 7개) | 수정 (한글 + 영문, 5개) |
|------------------|---------------------|
| LinearityRatio | **마우스 직선도 (linearityRatio)** |
| TremorStdDevb | **손떨림 표준편차 (tremorStdDev)** |
| dwellTime | **머뭇거림 시간 (dwellTime)** |
| pathRatio | **경로 비율 (pathRatio)** |
| avgVelocity | **평균 속도 (avgVelocity)** |
| ~~totalDist~~ | **제거** — 중간 계산값 (행동 지표 아님) |
| ~~botRisk~~ | **제거** — 최종 위험 점수 (행동 지표가 아니라 결과값) |

### 수정 이유

- 영문만 나열하면 비개발 청중이 이해 불가 — **한글(영문)** 병기
- `totalDist`는 feature 계산의 중간 변수, `botRisk`는 최종 산출 점수 — 둘 다 **입력 행동 지표가 아님**
- 실제 행동 feature는 **5가지** (docs 기준: `04-risk-scoring.md`)

# 5. 공격 시나리오를 방어 구조가 어떻게 받아내는가

앞서 설명한 공격자 우회 시도가 문서의 어느 방어 축에서 받아들여지는지를 개념적으로 매칭한다.
핵심은 공격이 달라져도 방어가 보는 축은 크게 변하지 않는다는 점이다.

## 5.1 매핑 원칙

**공격은 진화해도, 방어가 보는 것은 결국 행동과 상태다.**

공격자가 어떤 전략을 쓰더라도, 브라우저에서는 결국 마우스가 움직이고, 클릭이 발생하고, 세션 상태가 쌓인다.
방어는 공격자의 의도를 읽는 것이 아니라, 관찰 가능한 행동과 세션 맥락을 읽는다.

이때 방어의 기본 완충 구조는 세 단계다.

| 단계 | 역할 |
|------|------|
| 지연 | 공격 속도와 효율을 낮춤 |
| 챌린지 | 사람 여부를 재검증 |
| 보호 대기 | 통과 직후에도 바로 정상으로 풀어주지 않음 |

이 과정에서 남는 흔적은 사후 검토와 정책 최적화의 입력이 되어, 장기적으로 방어력 개선으로 이어진다.

## 5.2 공격 → 방어 매핑표

| # | 공격자 행위 | 주 방어 레이어 | 어떻게 받아내는가 |
|---|------------|--------------|-----------------|
| 1 | 순간이동 마우스 클릭 | 점수 계산 | 직선성, 경로 비율, 머뭇거림이 극단치로 나타나 내부 점수 급상승 → 빠르게 T3 근처로 이동 |
| 2 | 인간형 곡선 궤적 합성 | 점수 계산 | 형태는 완화돼도 속도·머뭇거림이 숙련자 고정값이면 누적으로 등급 상승 |
| 3 | 짧게 이동 후 텔레포트 섞기 | 점수 계산 + 챌린지 과정 판정 | 하이브리드 패턴이 속도·경로 비율 왜곡으로 드러남 |
| 4 | 고가치 좌석 반복 클릭 | 증거 축적 | 단시간 고가치 클릭 폭주 규칙 적중 |
| 5 | 과도한 조회 스캐닝 | 증거 축적 | 짧은 윈도우 내 조회 빈도 기준 초과 |
| 6 | 챌린지 정답 자동 제출 | 챌린지 2중 게이트 | 정답은 맞춰도 풀이 과정 궤적이 비인간적이면 차단 |
| 7 | 챌린지 반복 실패 후 재시도 | 증거 축적 + 챌린지 모듈 | 연속 실패 카운트, 재시도 제한, 임시 잠금 |
| 8 | 챌린지 통과 직후 다시 봇처럼 행동 | 보호 대기 + 히스테리시스 | 통과 직후 즉시 정상 등급으로 복귀하지 않음 |
| 9 | 스웜 동시 투입 | 세션 독립 판단 | 스웜 여부를 몰라도 각 세션의 신호를 독립 누적해 판단 |
| 10 | LLM 코디네이터로 전략 동적 변경 | 결정론 런타임 | 공격 측 LLM 판단은 방어 로직을 흔들지 못함 |
| 11 | 좌석 확보 실패 반복 | 증거 축적 | 실패 카운터와 4xx 패턴이 이유 코드로 누적 |
| 12 | 세션 재사용·토큰 교체 | 세션 상태 관리 | 같은 세션 식별자면 이전 위험을 이어받음 |
| 13 | 브라우저 지문 위장 | 점수 계산 + 챌린지 판정 | 지문이 아닌 실제 움직임과 타이밍이 주 탐지축 |
| 14 | 새로운 우회 패턴 등장 | 오프라인 최적화 + 사후 검토 | 온라인에서는 완충, 오프라인에서 정책 조정 후보 생성 |
| 15 | 순간 집중 공격 후 잠잠해짐 | EWMA + 비활성 감쇠 | 즉시 초기화되지 않고 서서히 위험이 감소 |
| 16 | 오픈 시각 완벽 동시 진입 | 세션 분리 + 독립 누적 | 요청은 분리되지만 세션별 점수 누적은 계속 작동 |
| 17 | 추천 모드로 넓은 점유 시도 | 등급 + 사후 검토 | 즉시 전부 막지 못해도 지연·챌린지와 사후 검토로 압박 |
| 18 | 특정 구역 반복 스나이핑 | 증거 축적 + 정책 조정 | 반복 시도 흔적이 누적되고, 향후 정책 조정 근거가 됨 |

## 5.3 구조적으로 봐야 할 세 축

이 매핑에서 중요한 포인트는, 방어가 단일 신호 하나에 의존하지 않는다는 점이다.

| 축 | 보는 것 | 담당 |
|----|---------|------|
| 행동 축 | 마우스 궤적, 타이밍, 속도, 떨림 | 점수 계산 |
| 패턴 축 | 단시간 반복 클릭, 조회, 실패 | 증거 축적 |
| 상태 축 | 누적된 세션 맥락, 통과 여부, 보호 대기 | 상태 관리 + 정책 적용 |

즉 공격자가 한 축을 어느 정도 흉내 내더라도, 다른 축에서 흔적이 남을 가능성이 크다.
이 세 축을 동시에 모두 자연스럽게 맞추는 것은 현실적으로 훨씬 어렵다.

## 5.4 시나리오 예시 — 한 문장씩 보면

| 사용자 유형 | 흐름 |
|------------|------|
| 정상 사용자 | 자연스러운 마우스와 적당한 머뭇거림으로 T0 유지 → 대부분 그대로 통과 |
| 단순 매크로 | 첫 구간부터 직선성·속도 극단치 → 빠르게 상승 → 챌린지 강제 → 과정까지 비인간적이면 차단 |
| 지능형 AI 에이전트 | 형태는 인간형이지만 속도·머뭇거림이 극단값에 고정 → 누적으로 등급 상승 → 챌린지와 보호 대기로 압박 |
| LLM 코디네이터 스웜 | 전략은 바뀌어도 세션별 행동과 상태는 남음 → 실시간 완충 + 사후 검토 대상 확대 |
